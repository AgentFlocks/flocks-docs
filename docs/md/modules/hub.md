# Flocks Hub 插件广场

Flocks Hub 是 Flocks 在 WebUI 中提供的插件广场。它的职责不是直接执行任务，而是把平台可复用能力以插件包的形式集中展示、筛选、预览、安装、更新和卸载，帮助团队把零散能力变成可管理的模块资产。

## 功能定位

Flocks Hub 更接近“能力分发与生命周期管理中心”，核心价值主要体现在四点：

- **统一入口**：把 Skill、Agent、Tool、Device、Workflow 五类插件放进同一个页面管理
- **可视化筛选**：支持按类型、使用场景、Tag、状态搜索和过滤
- **安装前预览**：安装前就能查看 Manifest、文件树、依赖、权限、版本等信息
- **安装后生效**：插件安装到本机插件目录后，可进入对应模块继续使用

## 支持的插件类型

当前 Hub 页面支持统一浏览以下五类插件：

- **Skill**：方法论、规范、检查清单、任务模板
- **Agent**：带 Prompt、工具集和角色边界的专家智能体
- **Tool**：API、本地脚本、MCP 等执行能力封装
- **Device**：偏设备接入类的插件包，可理解为更贴近安全产品接入的能力插件
- **Workflow**：可视化工作流和自动化剧本

## 操作步骤（WebUI）

### 步骤 1：进入 Flocks Hub

侧栏 **Agent 工作室 → Flocks Hub / 插件广场**。

页面顶部提供：

- 搜索框：按名称、描述、Tag、使用场景检索
- 视图切换：表格视图 / 目录视图
- 刷新按钮：重新读取当前本地可见的 Hub 目录内容

### 步骤 2：按条件筛选插件

Hub 支持按以下维度筛选：

- **类型**：Skill / Agent / Tool / Device / Workflow
- **使用场景**：例如告警研判、集成接入等
- **Tag**
- **状态**：可安装、已安装、可更新、不兼容等

### 步骤 3：查看插件详情

点击某个插件后，右侧会展开详情面板。常见可见信息包括：

- **概览**：ID、类型、Tag、使用场景、信任等级、Manifest 路径
- **文件**：插件文件树和可预览文件内容
- **依赖**：依赖的 Skills、Tools、Python 组件或外部依赖信息
- **权限**：工具权限、网络权限、Shell 权限、文件系统访问范围
- **版本**：打包版本、已安装版本、来源路径、安装位置

对 Workflow 类型插件，还会额外显示**流程图视图**，可直接在 Hub 中预览 `workflow.json` 对应的流程结构。

### 步骤 4：执行安装 / 更新 / 卸载

Hub 会根据插件当前状态显示操作按钮：

- **安装**：把当前插件安装到本机插件目录
- **更新**：用打包版本覆盖本地旧版本
- **卸载**：移除用户自行安装的本地副本

安装后，对应运行时会自动刷新。比如：

- Skill / Agent 会刷新缓存与发现结果
- Tool / Device 会刷新工具注册表和接入描述
- Workflow 会重新扫描可用工作流

### 安装前为什么要看权限和依赖

Hub 的详情页会暴露插件的权限与依赖信息，这一点很重要。对于安全运营场景，安装一个插件不只是“多了个功能”，也意味着它可能：

- 需要访问额外工具
- 允许网络请求
- 允许 Shell 执行
- 需要读写本地文件

因此建议在正式环境中优先检查：

1. 它依赖了哪些工具或外部能力
2. 它是否声明了网络 / Shell / 文件系统权限
3. 它的来源、版本和信任等级是否符合预期

### 目录视图与表格视图

Hub 提供两种浏览方式：

- **表格视图**：适合按类型、状态、Tag 快速筛选和批量查看
- **目录视图**：适合从插件包结构出发，按目录层级理解一个插件来自哪里、包含哪些包


## 案例

例如 `onesig_v2_5_3_D20250710` 这类 API 工具插件，就可以通过 Flocks Hub 统一浏览、安装和更新。用户先在 Hub 中查看其描述、依赖与权限信息，确认无误后安装，再到[工具清单](/md/modules/tools)中完成启用、测试和实际调用。

## 使用建议

- **先筛选再安装**：避免把 Hub 当成“看到就装”的列表页
- **先看权限和依赖**：尤其是涉及 Shell、网络和文件写入的插件
- **Workflow 先看流程图**：确认编排逻辑是否符合你的运营习惯
- **安装后去对应模块验证一次**：例如 Skill 让 Rex 真正加载一次，Workflow 跑一遍测试
