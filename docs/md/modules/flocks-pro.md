# Flocks Pro

Flocks Pro 是 Flocks 的 Pro 升级与授权管理模块。它用于在本地完成云账号登录、升级申请、审批状态跟踪、Pro 组件安装、License 同步和授权状态查看。

## 1. 功能定位

Flocks Pro 页面回答的是"**如何从 OSS 版升级并管理 Pro 授权**"。

它主要包含：

- 云账号登录。
- 提交 Flocks Pro 升级申请。
- 查看申请状态和历史申请。
- 下载并安装 Pro bundle。
- 查看 License 状态、License ID、到期时间、剩余天数和授权额度。
- 同步授权信息。
- 查看机器指纹和安装 ID。

## 2. Pro 与 OSS 的功能差异

Flocks OSS 适合单用户、本地化和社区场景；Flocks Pro 面向团队协作、企业治理和授权管理场景。两者核心差异如下：

| 能力 | OSS 版 | Pro 版 |
| --- | --- | --- |
| 账号体系 | 单个本地 `admin` 账号。 | 多账号体系，支持 `admin` / `member` 角色。 |
| 用户数据隔离 | 单账号使用统一会话和 Workspace。 | 每个用户拥有自己的对话历史和私有 Workspace 目录。 |
| 账号管理 | 支持登录、退出和修改密码。 | 支持创建用户、修改角色、重置密码、删除用户和配额管理。 |
| 审计日志 | 不提供审计日志页面。 | 提供账号、授权和关键系统操作审计。 |
| 授权管理 | 无 License 管理。 | 支持 License 状态查看、同步、到期时间和授权额度管理。 |
| 升级管理 | 使用 OSS 更新流程。 | 支持 Pro 申请、审批状态跟踪和 Pro 组件安装。 |

需要注意的是，Pro 多账号主要隔离个人对话历史和个人 Workspace 目录。Agent、Skills、Tools、设备、任务、Workflow、模型和通道等平台能力默认仍是团队共享能力，便于统一维护和复用。

## 3. 云账号登录

升级前需要先完成云账号登录。页面会从本地发起登录，跳转到云端登录，完成后回跳本地确认。

登录状态包括：

- 未登录。
- 已登录云账号。
- 当前会话缺少云账号信息，需要重新登录。

完成云账号登录后，才能提交升级申请。

## 4. 升级申请

### 4.1 申请类型

升级申请可以选择：

- 30 天试用。
- POC 授权。
- 正式授权。

申请表通常需要填写公司、申请人姓名，也可以补充邮箱、电话和申请说明。

### 4.2 申请状态

申请状态包括：

| 状态 | 说明 |
| --- | --- |
| `pending` | 待审批。 |
| `reviewing` | 审批中。 |
| `approved` | 已通过，可以开始升级。 |
| `activated` | 已激活。 |
| `rejected` | 已拒绝。 |
| `cancelled` | 已撤回。 |
| `expired_unactivated` | 已超时未激活。 |

如果已有申请在处理中，需要刷新状态或撤回后再发起新申请。

## 5. 开始升级

申请通过后，可以点击开始升级。升级过程会下载 Pro bundle，只安装 `flockspro` 组件，不升级 OSS core。

典型阶段包括：

1. 下载 Pro bundle。
2. 备份。
3. 应用更新。
4. 安装 Flocks Pro。
5. 构建。
6. 重启服务。
7. 完成。

升级安装后页面会等待服务重启。若重启超时，可以稍后手动刷新页面确认状态。

## 6. License 管理

### 6.1 查看授权状态

Pro 安装并激活后，页面会显示：

- Flocks Pro 版本。
- License 状态。
- License ID。
- License 到期时间。
- 剩余天数。
- 授权额度，例如管理员数量和成员数量。
- 授权信息同步时间。
- 安装 ID 和机器指纹。

### 6.2 同步授权信息

点击同步授权信息会刷新 License 状态。License 操作会写入 [审计日志](/md/modules/audit-logs)。

### 6.3 License 状态

常见 License 类型包括：

- 试用。
- 测试授权。
- 商业授权。

常见状态包括：

- 有效。
- 已吊销。
- 已到期。
- 已替换。

当 License 已失效时，可以重新提交 Flocks Pro 申请。

## 7. Pro 功能

Flocks Pro 启用后会开放更多企业级能力，典型包括：

- [账号管理](/md/modules/accounts)：多账号、角色、配额和密码重置。
- [审计日志](/md/modules/audit-logs)：管理员操作、授权操作和关键事件审计。
- 用户私有对话历史和私有 Workspace 目录。
- License 授权额度管理。
- Pro 组件升级和同步。

具体可用能力以当前 License 和部署版本为准。

## 8. 常见问题

### 8.1 为什么不能提交升级申请？

常见原因：

- 未完成云账号登录。
- 已有升级申请在处理中。
- 必填字段未填写。
- 网络或云端审批接口不可用。

### 8.2 Pro 升级会升级 OSS core 吗？

不会。当前升级流程下载并安装 Pro bundle 中的 `flockspro` wheel，不升级 OSS core。

### 8.3 Pro 功能为什么不可用？

优先检查：

- Pro 组件是否安装完成。
- License 是否有效。
- 授权是否已同步。
- 当前用户是否为管理员。
- 对应功能是否受 License 限制。
